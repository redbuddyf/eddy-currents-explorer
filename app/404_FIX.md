# 🔧 404 Error Fix Guide

## Error: "This page can't be found" or 404

**Good news:** Your DNS is working (domain is reaching GitHub)  
**Bad news:** GitHub isn't serving your site yet

---

## 🔍 Most Common Causes & Fixes

### **Cause 1: Wrong Repository Name**

Your repository MUST be named exactly:
```
eddy-currents-explorer
```

**Check right now:**
1. Go to https://github.com/YOUR_USERNAME
2. Look at your repository name
3. Is it named exactly `eddy-currents-explorer`?

**If it's named something else:**
- Repository → Settings tab
- Repository name field
- Change to: `eddy-currents-explorer`
- Click Rename
- **Wait 5 minutes** and try your domain again

**Common mistakes:**
- ❌ `eddy-currents` (missing "-explorer")
- ❌ `Eddy-Currents-Explorer` (wrong capitalization)
- ❌ `eddy_currents_explorer` (underscores instead of dashes)
- ✅ `eddy-currents-explorer` (exactly this)

---

### **Cause 2: GitHub Pages Not Actually Enabled**

**Check right now:**

1. Go to your repository on GitHub
2. Click **Settings** tab
3. Left sidebar → **Pages**
4. What do you see?

**Should show:**
```
🟢 Your site is live at https://eddy-currents.com
```

**If you see red error or nothing:**
- Source: Select **"Deploy from a branch"**
- Branch: Select **"main"**
- Folder: Select **"/ (root)"**
- Click **Save**
- Wait 5 minutes

---

### **Cause 3: Files in Wrong Location**

Your files must be in the ROOT of the repository, not in a folder.

**Wrong structure:** ❌
```
eddy-currents-explorer/
  └── eddy-currents-app/     ← DON'T DO THIS
      ├── index.html
      └── styles.css
```

**Correct structure:** ✅
```
eddy-currents-explorer/
  ├── index.html              ← Directly here
  ├── styles.css
  ├── app.js
  └── manifest.json
```

**How to check:**
1. Go to https://github.com/YOUR_USERNAME/eddy-currents-explorer
2. Do you see `index.html` directly, or do you see a FOLDER?
3. If you see a folder, your files are in the wrong place

**Fix:**
1. Click into the folder
2. Click each file
3. Click the **trash icon** to delete
4. Go back to main repository page
5. Click **"Add file"** → **"Upload files"**
6. Upload ALL files directly (not in a folder)

---

### **Cause 4: No index.html File**

GitHub Pages **REQUIRES** an `index.html` file in the root.

**Check:**
1. Repository main page
2. Is there a file called `index.html`?

**If missing:**
- Upload `index.html` to the repository root
- Wait 2 minutes
- Refresh your domain

---

### **Cause 5: Using HTTP Instead of HTTPS**

Try both:
```
https://eddy-currents.com    ← Try this first
http://eddy-currents.com     ← Should redirect to HTTPS
```

Always use `https://`

---

### **Cause 6: Custom Domain Not Saved in GitHub**

**Check:**
1. Repository → Settings → Pages
2. Is `eddy-currents.com` showing in the "Custom domain" field?
3. Did you click **Save**?

**If not there:**
1. Enter: `eddy-currents.com`
2. Click **Save**
3. Wait for DNS check to complete

## 🔥 NUCLEAR FIX (Start Over)

If nothing works, start fresh:

### Step 1: Delete Repository
1. GitHub repo → Settings tab
2. Scroll to bottom → "Delete this repository"
3. Type: `YOUR_USERNAME/eddy-currents-explorer`
4. Click "I understand..."

### Step 2: Create New Repository
1. GitHub homepage → Click **"New"**
2. Repository name: `eddy-currents-explorer` (exactly!)
3. Make it **Public**
4. Click **Create repository**

### Step 3: Upload Files
1. In new repo, click **"uploading an existing file"**
2. Drag ALL files from `eddy-currents-app/` folder:
   - index.html
   - styles.css
   - app.js
   - manifest.json
   - sw.js
3. Click **"Commit changes"**

### Step 4: Enable Pages
1. Settings → Pages
2. Source: "Deploy from a branch"
3. Branch: "main"
4. Click **Save**

### Step 5: Add Custom Domain
1. Still in Pages settings
2. Custom domain: `eddy-currents.com`
3. Click **Save**
4. Check "Enforce HTTPS"

### Step 6: Wait
Wait 5-10 minutes, then test:  
`https://eddy-currents.com`
